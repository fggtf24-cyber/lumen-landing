# Lumen — бэкенд для формы заявок

Хендофф для бэкенд-разработчика. Здесь всё, чтобы быстро въехать и начать.

---

## TL;DR

Фронтенд — **статический лендинг** (`../index.html`), задеплоен на GitHub Pages:
<https://fggtf24-cyber.github.io/lumen-landing/>

Единственная динамическая часть — **форма заявки** в секции «Контакты». Сейчас
она обрабатывается только на клиенте (показывает «Заявка принята», но никуда не
отправляет). Задача бэкенда — **принять заявку и доставить её** нам в Telegram
(`@fggtf24`) и/или на почту (`fggtf24@gmail.com`).

В этой папке лежит **готовый референс на Node/Express** — можно запустить за пару
минут, доработать или переписать на своём стеке. Контракт API важнее реализации.

---

## Что нужно сделать

Поднять один эндпоинт `POST /api/lead`, который:

1. принимает JSON с полями формы;
2. валидирует их;
3. отсекает спам (honeypot + rate-limit);
4. доставляет заявку (Telegram Bot API и/или email по SMTP);
5. отвечает `{ "ok": true }` или ошибкой.

Хранение в БД **не требуется** (лиды приходят в мессенджер/почту). Если захотите
дублировать в базу/CRM — welcome, но это опционально.

## Контракт API

### `POST /api/lead`

Запрос (`Content-Type: application/json`):

```json
{
  "name": "Иван",
  "contact": "@ivan или ivan@mail.ru или +7...",
  "message": "Нужен лендинг для кофейни...",
  "company": ""
}
```

| Поле      | Обяз. | Правила                                  |
|-----------|-------|------------------------------------------|
| `name`    | да    | 2–80 символов                            |
| `contact` | да    | ≥ 3 символов (телефон / e-mail / @tg)    |
| `message` | нет   | ≤ 4000 символов                          |
| `company` | —     | **honeypot**: всегда пустое; если пришло непустым — это бот, тихо игнорируем |

Ответы:

- `200 { "ok": true }` — принято;
- `400 { "ok": false, "errors": ["name", "contact"] }` — ошибка валидации;
- `429` — превышен лимит запросов;
- `500 { "ok": false, "error": "delivery_failed" }` — не удалось доставить.

### `GET /health`
`200 { "ok": true }` — для healthcheck-ов хостинга.

## Где форма во фронте

Файл `../index.html`, секция `#contact`:

- `<form id="contactForm">` с полями `name`, `contact`, `message`;
- обработчик отправки — в `<script>` внизу файла, функция на `submit` формы
  (сейчас просто `e.preventDefault()` + подстановка блока `.form-success`).

Чтобы подключить бэкенд, замените текущий обработчик на вариант из
[`frontend-snippet.js`](./frontend-snippet.js) и добавьте honeypot-поле
(инструкция там же). Больше на фронте менять нечего.

## Запуск референс-сервера

Нужен Node.js 18+ (используется встроенный `fetch`).

```bash
cd backend
cp .env.example .env      # заполнить значения
npm install
npm start                 # поднимет http://localhost:3000
```

Быстрая проверка — см. [`requests.http`](./requests.http) (VS Code REST Client)
или curl:

```bash
curl -X POST http://localhost:3000/api/lead \
  -H "Content-Type: application/json" \
  -d '{"name":"Тест","contact":"@fggtf24","message":"проверка"}'
```

## Переменные окружения

Все в `.env` (см. `.env.example`). Ничего секретного в репозиторий не коммитим —
`.env` уже в `.gitignore`.

| Переменная            | Зачем                                                        |
|-----------------------|--------------------------------------------------------------|
| `PORT`                | порт сервера (по умолчанию 3000)                             |
| `ALLOWED_ORIGINS`     | CORS-белый список, через запятую. Прод: `https://fggtf24-cyber.github.io` |
| `TELEGRAM_BOT_TOKEN`  | токен бота от @BotFather                                     |
| `TELEGRAM_CHAT_ID`    | chat_id, куда слать заявки                                   |
| `SMTP_HOST/PORT/USER/PASS` | SMTP для email (опционально)                           |
| `LEAD_EMAIL_TO`       | куда слать письма (`fggtf24@gmail.com`)                      |

Достаточно настроить **любой один** канал (Telegram *или* email). Если не заданы
переменные канала — он просто пропускается.

### Настроить Telegram-бота (быстрее всего)

1. В Telegram напишите **@BotFather** → `/newbot` → получите `TELEGRAM_BOT_TOKEN`.
2. Напишите своему новому боту любое сообщение (чтобы он мог вам отвечать).
3. Узнайте `chat_id`: откройте
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите `chat.id`
   (или добавьте бота в группу и возьмите id группы).
4. Впишите токен и chat_id в `.env`.

### Email через SMTP

Для Gmail это App Password (обычный пароль не подойдёт при 2FA). Либо возьмите
любой транзакционный SMTP (Resend/Postmark/Mailgun/Yandex).

## Деплой

Сайт статический на GitHub Pages, поэтому API живёт **отдельно**. Варианты:

- **Serverless** (проще всего): вынести логику `/api/lead` в функцию на
  Vercel / Netlify / Cloudflare Workers. `server.js` легко адаптируется —
  тело обработчика переносится в функцию.
- **Обычный хост**: Render / Railway / Fly.io / небольшой VPS — деплой `server.js`
  как есть.

После деплоя:

1. пропишите публичный origin сайта в `ALLOWED_ORIGINS`
   (`https://fggtf24-cyber.github.io`);
2. подставьте URL API во фронтовый сниппет (`LEAD_ENDPOINT`);
3. проверьте, что заявка реально доходит.

> **CORS**: сайт на домене `fggtf24-cyber.github.io` шлёт запрос на ваш API —
> браузер потребует правильные CORS-заголовки. В референсе это уже настроено
> через `ALLOWED_ORIGINS`.

## Безопасность (уже учтено в референсе)

- Валидация и лимит размера тела (16 КБ).
- **Rate-limit**: 5 запросов/мин с IP.
- **Honeypot** `company`: скрытое поле, боты его заполняют — мы молча дропаем.
- CORS-белый список.
- Секреты только в `.env` (в репозиторий не попадают).

Опционально можно добавить: капчу (hCaptcha/Turnstile), логирование в БД,
уведомление об ошибках.
